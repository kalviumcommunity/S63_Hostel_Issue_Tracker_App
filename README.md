# S63_Hostel_Issue_Tracker_App

Hostel Issue Tracker
Overview

Hostel Issue Tracker is a mobile application that helps hostel residents report and track issues such as maintenance problems, cleanliness concerns, or facility complaints. The app allows users to submit issues easily and enables administrators to monitor, manage, and resolve them efficiently.

Features

Residents can report hostel-related issues.
Real-time issue tracking and status updates.
Admin panel to manage and resolve complaints.
Simple and user-friendly interface.

Tech Stack

Frontend: Flutter
Backend: Firebase
Database: Firebase Firestore
Authentication: Firebase Authentication

Purpose

The goal of this project is to improve communication between hostel residents and management by providing a transparent and organized system for reporting and resolving issues.

Tech Stack Explained
Flutter (Frontend)

Flutter is used to build the user interface of the application. It is a cross-platform framework developed by Google that allows developers to create mobile apps for Android and iOS using a single codebase. In this project, Flutter is responsible for displaying screens such as issue submission, issue list, and status tracking while providing a smooth and responsive user experience.

Firebase (Backend)

Firebase acts as the backend service for the application. It provides ready-to-use tools for handling authentication, database management, and real-time data updates without requiring a traditional server setup. Using Firebase helps the app scale easily and simplifies backend development.

Firebase Firestore (Database)

Firestore is the cloud database used to store all application data. It stores information such as user details, reported issues, issue descriptions, timestamps, and issue status. Firestore supports real-time updates, which allows users to see changes in issue status immediately.
